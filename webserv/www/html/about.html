Qu'est-ce que Webserv?

Webserv est un serveur HTTP écrit en C++98 conforme à la plupart des spécifications du sujet 42. Il supporte:

  • ✅ GET, POST, DELETE
  • ✅ Configuration multi-serveur NGINX-like
  • ✅ Locations avec règles par route
  • ✅ Upload automatique de fichiers
  • ✅ Exécution CGI (PHP, Python)
  • ✅ Pages d'erreur personnalisées
  • ⏳ Redirections HTTP
  • ⏳ Directory listing (autoindex)

Architecture

Webserv utilise une boucle événementielle poll-based pour gérer tous les I/O de façon non-bloquante. Les clients sont maintenus dans une map avec état (requête incomplète, réponse en attente).

Configuration

Le serveur est configuré via un fichier config au format inspiré de NGINX: 

server {
    listen 8081;
    server_name webserv;
    root /;
    
    location / {
        root /www/html;
        limit_except GET POST;
        autoindex true;
    }
    
    location /uploads {
        limit_except POST;
        upload_path /uploads;
    }
}

Points Clés du Sujet

  • Non-bloquant obligatoire: Tous les I/O passent par poll(), jamais de read/send direct
  • C++98 strict: Pas de C++11, C++17 ou STL moderne
  • Pas de fork() sauf CGI: Un seul process serveur, fork uniquement pour exécuter scripts
  • Résilience: Le serveur continue même si un client meurt ou une location est invalide
